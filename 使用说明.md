# QMS质量分析平台使用说明

## 推荐：在其他电脑运行

将以下压缩包复制给对方：

`release\QMS质量分析平台-免安装版.zip`

对方电脑操作：

1. 解压整个压缩包，不能只把EXE单独拖出来。
2. 打开“QMS质量分析平台-免安装版”文件夹。
3. 双击“QMS质量分析平台.exe”。
4. Windows首次提示未知发布者时，选择“更多信息”→“仍要运行”。

不需要安装Node.js、Office或开发环境。支持Windows 10/11 64位。

## 本地数据保存

- 导入后的解析数据自动保存到当前电脑。
- 关闭或重新打开软件，会自动恢复上次的数据源。
- 数据只保存在当前电脑，不上传外部服务器。
- 浏览器版和桌面版使用各自的本地数据空间，互不自动同步。
- 把软件复制到另一台电脑时，只复制程序，不会携带本机已导入的质量数据；新电脑需首次导入。

## 数据源管理

左侧进入“数据导入”，可以分别管理IQC、IPQC、OQC、DQA。

- “导入/替换”：只允许导入当前模块识别成功的文件。
- 相同模块且文件名相同：自动替换旧数据，避免重复统计。
- 不同文件名：作为新的数据源增加。
- “删除”：删除指定数据源，并立即重新计算全部分析。

建议更新源数据时保持原文件名不变，这样系统会直接替换。

## 网页开发版

双击“启动网页版.cmd”，访问：

- 经营驾驶舱：`http://127.0.0.1:4173/#executive`
- 质量工作台：`http://127.0.0.1:4173/#workspace`

## 重新生成免安装版

在项目目录运行：

```powershell
npm.cmd install
npm.cmd run desktop:pack
```

生成结果位于`release`目录。
# 项目体积说明

日常网页版开发不需要 Electron 桌面运行包。为避免项目持续膨胀，npm 缓存不再保存在项目目录内，Vite 缓存可以随时删除。

最终需要打包桌面版前，在项目目录运行一次：

```powershell
npm.cmd install
```

该命令会恢复已从日常开发目录中清理的 Electron 打包依赖。
